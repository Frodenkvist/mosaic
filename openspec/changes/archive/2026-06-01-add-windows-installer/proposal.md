## Why

Mosaic can currently only be run from source (`dotnet run`); there is no way to ship it to an end user. There is no installer, no Start Menu entry, no Apps & features registration, no clean uninstall, and no installed version visibility. We need a wizard-driven installer `.exe` and a matching uninstaller `.exe` so Mosaic can be distributed and removed like a normal Windows desktop app — and so that a future auto-update feature has a foundation to build on.

## What Changes

- Add a Windows packaging build that publishes Mosaic as a **self-contained, win-x64** app (the .NET 10 Desktop runtime is bundled, so the target PC needs no prerequisites).
- Produce a **wizard-driven installer** `MosaicSetup-<version>.exe` (Inno Setup) with the standard pages: welcome → install location → shortcut/options → installing → finish (with an optional "Launch Mosaic" checkbox).
- Install **per-user** (no admin / no UAC) to `%LocalAppData%\Programs\Mosaic`, so future silent auto-updates won't require elevation.
- Create a **Start Menu** shortcut and an **optional desktop** shortcut.
- Register the app in **Apps & features** (Add/Remove Programs) with name, version, publisher, icon, and an uninstall entry.
- Ship a **standalone uninstaller** `.exe` (auto-generated by the installer toolchain) that removes program files and shortcuts.
- On uninstall, **offer to also delete user data** (game library DB, artwork cache, settings under `%LocalAppData%\Mosaic`); default to **keeping** it so reinstall/upgrade preserves the library.
- Handle **in-place upgrade**: installing over an existing version replaces program files without losing user data.
- Add **assembly/product version metadata** to `Mosaic.csproj` so the installed product reports a version.
- Document the publish + package build commands.

Out of scope for this change (future, design will not preclude): **auto-update** — the app checking for and applying new versions itself.

## Capabilities

### New Capabilities
- `installer`: building, distributing, installing, upgrading, and uninstalling Mosaic on Windows — covering the self-contained publish, the wizard installer behavior, per-user install location, shortcuts, Apps & features registration, the standalone uninstaller, and user-data handling on removal.

## Impact

- **New build tooling** under `installer\`: an Inno Setup script (`Mosaic.iss`) and a packaging script (`package.ps1`) that publishes the app and invokes the Inno Setup compiler (`ISCC.exe`).
- **`Mosaic.csproj`**: add `Version`/`AssemblyVersion`/`FileVersion`, `RuntimeIdentifier`/`SelfContained`/publish properties (or pass via the publish command), and product/company metadata.
- **Build-time dependency**: Inno Setup (`ISCC.exe`) — not a runtime dependency of the app.
- **Output artifact**: `MosaicSetup-<version>.exe` (a single distributable file).
- **No app runtime behavior changes** are required to install; data continues to live under `%LocalAppData%\Mosaic` via `AppPaths` and the installer never touches it except on opt-in uninstall cleanup.
- **Docs**: add packaging/release commands to `CLAUDE.md`.
